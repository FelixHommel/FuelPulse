# ADR 001: Use Modular Core Architecture

## Status

*Accepted*

## Context

The Application should be extendable without modifying the application core. Future modules may collect and analyze
other time-series data besides fuel prices.

## Decision

The application core will expose extension points through interfaces. Functional behavior is implemented in modules that
register themselves with the application during startup.

## Consequences

### Pros

- Separation of concerns
- Easier testing
- Easy future extensions

### Cons

- Slightly higher initial complexity
