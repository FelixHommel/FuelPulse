# ADR 006: Separate Collection From Analysis

## Status

*Accepted*

## Context

Data collection and statistical analysis have different lifecycles. Collection is continuous and latency-sensitive,
while analysis is compute-oriented and may evolve independently over time.

## Decision

The collector is responsible only for acquiring and persisting raw measurements. It never computes statistics.
Analysis operates exclusively on persisted historical data and generates versioned reports.

## Consequences

### Pros

- Historical data can be reanalyzed with improved algorithms
- Analysis can be rerun without contracting the external API
- Collection remains lighweight and robust
- Enables offline experimentation and backfilling

### Cons

- Requires additional storage
- Reports are generated asynchronously rather than immediately
